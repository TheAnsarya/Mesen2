# Genesis M68000 CPU Integration

## Scope

Define safe integration boundaries for M68000 execution in a way that preserves interrupt and memory semantics.

## Findings

- Genesis Plus GX includes dedicated 68k core integration and memory map wiring in core startup and memory files.
- PicoDrive supports multiple 68k backend cores and explicit reset/interrupt callback glue.
- ares exposes an abstracted M68000 contract with clean read/write/idle/wait hooks.

## Nexen Guidance

1. Integrate M68000 with explicit callback hooks for bus and interrupt interactions.
2. Preserve instruction timing visibility for trace and validation tooling.
3. Avoid early micro-optimizations until baseline correctness and deterministic replay pass.
