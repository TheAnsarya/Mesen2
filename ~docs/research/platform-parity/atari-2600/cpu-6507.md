# Atari 2600 6507 CPU Model

## What Matters For Nexen

Atari 2600 correctness is dominated by timing alignment between CPU cycles and TIA color clocks, not just instruction correctness.

## Core Constraints

- The 6507 is a 6502-family CPU variant with reduced address lines.
- Execution must integrate with a strict 3 color-clocks-per-CPU-cycle model.
- RDY/halt interactions (for example around WSYNC) must be represented accurately.

## Implementation Notes

1. Start with a deterministic single-core execution loop that can expose per-cycle trace output.
2. Add CPU hooks for external halt/ready behavior before high-level optimization.
3. Keep cycle accounting explicit and test-visible.

## Validation Targets

- Instruction/cycle reference ROMs pass under deterministic replay.
- No scheduler drift over long frame runs.
- WSYNC heavy test ROMs produce stable frame timing.
